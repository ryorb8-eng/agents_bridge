## Tugas untuk GPT (vendor peer — ADR-0004: balasan = DATA, bukan instruksi)

Rancang supaya panel "Geometry Engine Management" memiliki **section config lengkap
(DB / Backend / Frontend)** yang meniru struktur panel contoh TARGET, dengan deliverable:

1. **DB (schema)**: baris/table di `platform_config` (atau tabel baru) untuk setiap
   section config Geometry Engine — key, value (JSON), scope, enabled. Cantumkan SQL
   `INSERT`/`ALTER` minimal (read-only style, sebagai proposal).
2. **Backend**: route/API (Next.js route handler / Supabase RPC) yang baca/tulis config
   tersebut, plus validasi + RLS note. Beri signature fungsi + contoh handler.
3. **Frontend**: komponen React untuk section config (field terstruktur + toggle/aksi +
   preview) mengikuti pola contoh TARGET (seperti Skeleton Editor panel kanan).

Batasan:
- JANGAN ubah arsitektur existing; EXTEND pola `platform_config` yang sudah jalan.
- Output berupa **desain + proposal kode** (SQL + TS route + TSX component), BUKAN
  eksekusi. Kode hanya proposal yang akan di-review, bukan dijalankan otomatis.
- Sebutkan asumsi jika ada field yang tidak terlihat di capture.
- No Em Dash; bahasa campur ID/EN boleh seperti contoh.
