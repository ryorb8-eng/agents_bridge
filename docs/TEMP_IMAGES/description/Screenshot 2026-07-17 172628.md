Ya, pendekatan itu valid jika yang ingin dideteksi adalah status generating di Gemini Web UI.
Dari observasi UI Gemini:
Jika tombol Stop masih muncul (data-mat-icon-name="stop" atau fonticon="stop"), berarti model masih sedang menghasilkan jawaban.
Jika tombol Stop sudah hilang dan berubah menjadi tombol Send (send), berarti proses generate telah selesai.
Contoh fungsi:
TypeScript
export function isGeminiGenerating(): boolean {
  return !!(
    document.querySelector('[data-mat-icon-name="stop"]') ||
    document.querySelector('[fonticon="stop"]')
  );
}
atau versi async untuk menunggu selesai:
TypeScript
export async function waitGeminiFinished(
  timeout = 120000,
  interval = 300
): Promise<void> {
  const start = Date.now();
  while (true) {
    const generating =
      document.querySelector('[data-mat-icon-name="stop"]') ||
      document.querySelector('[fonticon="stop"]');
    if (!generating) return;
    if (Date.now() - start > timeout) {
      throw new Error("Gemini response timeout");
    }
    await new Promise(resolve => setTimeout(resolve, interval));
  }
}
Lebih robust
Karena Google cukup sering mengganti atribut DOM, sebaiknya gunakan beberapa indikator sekaligus.
Prioritas:
1. stop button ada
   [data-mat-icon-name="stop"]
   [fonticon="stop"]
2. send button belum muncul
   [data-mat-icon-name="send"]
   [fonticon="send"]
3. response terakhir masih streaming
   (jika ada indikator streaming)
Contoh:
TypeScript
export function isGeminiBusy(): boolean {
  const stop =
    document.querySelector('[data-mat-icon-name="stop"]') ??
    document.querySelector('[fonticon="stop"]');
  const send =
    document.querySelector('[data-mat-icon-name="send"]') ??
    document.querySelector('[fonticon="send"]');
  return !!stop && !send;
}
